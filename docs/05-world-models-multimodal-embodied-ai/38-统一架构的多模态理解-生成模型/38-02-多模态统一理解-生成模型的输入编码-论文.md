# 38.2 多模态统一理解-生成模型的输入编码（论文）

> 本文是论文阅读笔记，内容代表对应论文方法或作者理解，不应直接视为领域共识或工程最佳实践。

## 一、主要编码方案

统一理解与生成模型要处理同一张图像时，通常会先加入一个视觉分词器。视觉分词器可以基于 VQ-VAE 一类架构，把连续图像像素空间压缩成可进行训练的离散空间。换句话说，系统会先把图像切分成一系列离散的视觉标记（Visual Tokens），再让这些视觉标记在格式和维度上与文本标记（Text Tokens）对齐。

### （一）分层视觉标记

一种常见思路是用分层残差方式逐级编码图像：

- **Level 1，$l=1$**：模型先用第一层密码本匹配原始图像特征。这一层捕捉的是最核心、最抽象的语义信息，可以理解为图像的目录。
- **计算残差**：第一层匹配并不精确，会产生残差，即原始特征减去第一层特征后的差值。
- **Level 2，$l=2$**：模型再用第二层密码本专门量化第一层留下的残差，第二层捕捉的是次一级的细节。
- **循环堆叠**：依此类推，直到第 8 层（$l=8$）。每一层都在修补上一层留下的微小误差，最终捕捉更细粒度的高频纹理和边缘信息。

统一模型中的输入通常分为两类：

- **文本输入**：直接通过语言模型自带的 Embedding 层查表，转化为文本 Token $X_{\text{text}}$。
- **图像输入**：图像会同时复制两份，并分别进入两个视觉编码器：
  - 喂给 ViT，输出代表高级概念的语义 Token $X_{\text{ViT}}$。
  - 喂给 VAE Encoder，输出代表连续空间结构的隐变量 Token $X_{\text{VAE}}$。

### （二）理解特征与生成特征的差异

在过去的统一多模态模型中，“看”和“画”实际上使用的是两套完全不同的特征语言：

- **视觉理解（看）**：大语言模型为了看懂图片，通常使用 CLIP 或 SigLIP 这类表征编码器。这些编码器提取出来的是高维、富含语义的特征，例如 1152 维。这就像是 LLM 的视觉母语。
- **视觉生成（画）**：传统扩散模型为了节省算力，使用 VAE 将像素级图像压缩成低维隐特征，例如几十维，再在这个低维空间里进行去噪生成。

这种分裂会带来一个直接问题：扩散模型生成的是 VAE 低维特征，而 LLM 认识的是 CLIP/SigLIP 的高维语义特征。如果 LLM 想检查扩散模型画得对不对，就必须走一条繁琐链路：

1. 扩散模型生成 VAE 特征。
2. VAE 解码器把这些特征还原成像素图片。
3. CLIP 编码器再把像素图片编码成高维语义特征。
4. LLM 再基于这些语义特征进行评估。

这导致模型内部仿佛存在两个不互通的大脑。

### （三）RAE 的输入编码思路

RAE 的核心思想非常直接：既然 LLM 习惯看 SigLIP 这类高维语义特征，就不要再让扩散模型生成 VAE 的低维语义特征，而是直接生成 SigLIP 的高维语义特征。

因此，在 RAE 的架构下，模型取消了传统的 VAE 编码器，只使用 SigLIP-2 作为唯一编码器：

- **统一生成空间**：扩散模型不再预测 VAE 的低维特征，而是直接在 $16 \times 16 \times 1152$ 维的 SigLIP-2 语义特征空间中生成。
- **无缝的模态互通**：扩散模型生成的结果本身就是 SigLIP-2 特征，也就是 LLM 的视觉语言。扩散模型刚生成完图像，LLM 不需要经过任何像素解码和重新编码，就能直接读取这些特征序列，并立即判断图像是否符合要求。

RAE 中的编码器和解码器承担不同角色：

- **重编码器（Heavy Encoder）**：直接白嫖开源的、大参数量的视觉基础模型，例如数十亿参数的 SigLIP-2、DINOv2 或 MAE。它的任务是深思熟虑地看透真实图像，提取出极其高维、富含语义的特征表示。
- **轻解码器（Light Decoder）**：只是一个层数很少、参数量很小的浅层 ViT。它只负责把高维特征矩阵降维并扩展成人类肉眼可见的 RGB 像素矩阵。

最终，RAE 不是回到了低级像素空间，而是把生成这件事推进到与理解同一级别的高维语义空间。

### （四）高维语义空间中的训练稳定性

现代 T2I 模型常采用“大规模预训练加高质量小数据微调”的范式，这会带来冷启动问题：

- **VAE 的脆弱性**：实验观察到，基于 VAE 的模型在高质量数据集上微调仅 64 个 Epoch 后，就会遗忘灾难性地过拟合。它的训练损失会迅速暴跌，模型开始死记硬背训练样本，并丧失泛化能力。
- **RAE 的稳健性**：RAE 即使微调到 256 个 Epoch 甚至 512 个 Epoch，性能依然保持坚挺。这说明 RAE 的高维语义空间起到了隐式正则化作用，强迫模型学习真正的特征分布，而不是记住训练集细节。

其中一个必须保留的核心机制是**维度感知噪声调度（Dimension-aware noise scheduling）**。因为高维隐空间与低维隐空间特性不同，模型需要根据有效数据维度 $m$ 和 Token 数乘以特征维度 $n$ 对时间步进行偏移缩放。时间步偏移公式为：

$$
t_m = \frac{\alpha t_n}{1 + (\alpha - 1)t_n}
$$

其中 $n$ 为基准维度，并且：

$$
\alpha = \frac{m}{n}
$$

## 二、Embedding 训练方法

视觉表征也常借助图文对比学习对齐图像和文本，其典型损失为：

$$
L_{\text{contrastive}} = -\log \frac{\exp(s(I,T)/\tau)}{\sum_j \exp(s(I,T_j)/\tau)}
$$

在 VQ 架构中，编码器输出连续向量 $z_e$，量化器从密码本中查找最接近的离散向量 $z_q$，最后解码器将 $z_q$ 还原为图像。最近邻量化可写为：

$$
k = \arg\min_j \lVert z_e - e_j \rVert_2
$$

$$
z_q = e_k
$$

梯度的关键问题在于 $\arg\min$ 函数的数学性质：

1. 大多数情况下，当前 $e_k$ 依然是最近项，索引 $k$ 不变，$z_q$ 也不变，此时导数为 0。
2. 极少数情况下，$z_e$ 恰好跨过边界，索引 $k$ 突然改变，此时导数为无穷大。

因此，模型通常使用直通估计器（Stop-Gradient / Straight-Through）让梯度绕过不可导的离散选择：

$$
z_q = z_e + \mathrm{sg}[e_k - z_e]
$$

解码器传回来的梯度不仅没有变成 0，反而以 1:1 的比例直接穿透量化层，交给编码器。

## 三、离散编码方案中的特殊训练方法

VQ 模型常加入承诺损失（Commitment Loss）约束编码器输出：

$$
\alpha \lVert U_G - \mathrm{sg}[Q] \rVert^2
$$

这里 $U_G$ 是编码器输出的连续潜变量，$Q$ 是离散化后的量化特征。$\mathrm{sg}[\cdot]$ 表示停止梯度操作。由于量化过程中存在信息丢失，这一项会迫使编码器输出的连续特征主动靠近被选定的离散特征 $Q$，避免连续特征在潜空间中随意游走。

密码本本身也需要更新，常见做法包括三类。

### （一）基于梯度的 Codebook Loss

在总损失中人为加入额外惩罚项，对每次前向传播中被选中的码本向量 $e_k$ 和对应编码器连续特征 $z_e$ 计算均方误差：

$$
L_{\text{codebook}} = \lVert \mathrm{sg}[z_e] - e_k \rVert_2^2
$$

这里的 $\mathrm{sg}[\cdot]$ 仍表示停止梯度。在这个损失项中，编码器输出 $z_e$ 被视为固定目标点，优化器只会更新码本向量 $e_k$，使其在连续空间中一步步靠近编码器输出。

为了防止编码器乱跑，通常还会配套加入对应的承诺损失：

$$
\lVert z_e - \mathrm{sg}[e_k] \rVert_2^2
$$

### （二）指数移动平均（EMA）

在实际训练大规模 VQ-VAE 或 VQ-GAN，包括 UniTok 底层时，使用梯度下降更新码本往往非常不稳定。因此更常用 EMA 来更新码本：

1. 在每个 Batch 中，记录哪些连续特征 $z_e$ 被分配给当前码本向量 $e_k$。
2. 计算这个 Batch 中分配给 $e_k$ 的所有 $z_e$ 的均值，记为 $\bar z$。
3. 用移动平均更新 $e_k$ 的权重：

$$
e_k^{(\text{new})} = \gamma \cdot e_k^{(\text{old})} + (1 - \gamma)\cdot \bar z
$$

其中 $\gamma$ 是衰减系数，例如 0.99。EMA 不需要计算梯度，相当于动态执行在线 K-Means，让码本向量能非常平滑、稳定地追踪编码器特征分布的变化。

### （三）UniTok 的特殊更新

UniTok 除了普通视觉特征，还需要让码本更贴近图像特征对应的语义对齐位置。在图文对比学习目标中，被激活的离散 Token 对应的码本向量 $e_k$ 会与对应文本连续向量计算余弦相似度。这个额外梯度不会只聚焦原图 Patch，还会推动码本向量移向与文本描述对齐的位置。

系统会定期扫描码本。如果某个向量 $e_k$ 使用频率低于阈值，就会直接将它抛弃，并从当前 Batch 的编码器特征 $z_e$ 中随机抽取一个活跃向量作为新的 $e_k$。这样做能避免死码本反复参与后续 EMA 或梯度更新。
